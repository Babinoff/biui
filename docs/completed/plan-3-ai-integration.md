# План реализации Этапа 3: Интеграция ИИ и Генерация кода

## Цель этапа
Организовать взаимодействие с LLM (Language Model) для генерации кода обработки данных на основе промтов пользователя. Создать интерфейс ввода задач, отображения и корректировки сгенерированного кода.

---

## Задачи этапа

### 1. Настройка API клиента
- [ ] Выбор провайдера LLM (OpenAI, Anthropic, Hugging Face или локальный сервер)
- [ ] Создание конфигурации API ключей
  - [ ] Поддержка `.env` файла для хранения API ключа
  - [ ] Проверка ключа при старте приложения
- [ ] Создать класс/модуль `LLMClient`
  ```typescript
  class LLMClient {
    async generateCode(prompt: string, context: Context): Promise<GenerationResult>;
  }
  ```

### 2. Контекст для генерации
- [ ] Формирование контекста из:
  - Схемы данных (названия колонок, типы значений)
  - Примера данных (первые N строк)
  - Истории преобразований (если есть предыдущие ноды)
- [ ] Создание шаблона промта
  ```typescript
  interface GenerationContext {
    schema: ColumnInfo[];
    sampleData: Record<string, any>[];
    previousTransforms?: string[];
  }
  ```

### 3. Генерация кода
- [ ] Реализовать метод `generateCode(prompt, context)` в ноде
- [ ] Парсинг ответа от LLM:
  - Извлечение блока кода (Markdown code block)
  - Валидация синтаксиса
- [ ] Отображение результата в кодовом поле ноды

### 4. Интерфейс взаимодействия с промтом
- [ ] Создать компонент `PromptEditor`:
  - Поле ввода промта (textarea/富文本)
  - Кнопка генерации кода
  - Индикатор статуса (loading/success/error)
- [ ] Кодовое поле с подсветкой синтаксиса (PrismJS / Highlight.js)
- [ ] Кнопка "Play" для тестового выполнения

### 5. Система корректировок (Chain of Prompts)
- [ ] Хранение цепочки промтов в истории
- [ ] Визуализация истории взаимодействия:
  ```
  Prompt 1 → Code 1
    ↓ (User correction)
  Prompt 2 → Code 2
  ```
- [ ] Возможность вернуться к предыдущей версии

### 6. Извлечение данных из кода
- [ ] Анализ переменных в сгенерированном коде:
  - Имя результирующего датафрейма/массива
  - Имена колонок (если известны)
- [ ] Автоматическое обновление метаданных ноды

### 7. Обработка ошибок и исключений
- [ ] Отображение ошибок выполнения кода
- [ ] Подсветка строк с ошибкой
- [ ] Предложение исправления на основе анализа сообщения об ошибке

```
src/

│   └── codeGeneration.ts      # Результат генерации кода
└── hooks/
    └── useCodeGeneration.ts   # Хук для управления процессом генерации
```

### Интерфейсы

```typescript
interface ColumnInfo {
  name: string;
  type: 'string' | 'number' | 'boolean' | 'date';
}

interface GenerationContext {
  schema: ColumnInfo[];
  sampleData: Record<string, any>[];
  previousTransforms?: string[];
}

interface GenerationResult {
  code: string;
  metadata: {
    outputColumns: string[];
    variablesUsed: string[];
  };
}
```

### Пример шаблона промта

```
Твоя задача — написать Python-код (pandas) для обработки данных.

Данные имеют следующую структуру:
{columns: ["date", "product", "amount"], types: ["str", "str", "int"]}

Пример данных:
| date       | product   | amount |
|------------|-----------|--------|
| 2024-01-01 | Product A | 1500   |

Выполни действие пользователя: {userPrompt}
```

## Критерии приемки Этапа 3

1. Приложение подключается к LLM API
2. Пользователь может ввести промт и получить код
3. Сгенерированный код отображается в редакторе с подсветкой
4. Реализована простая система корректировок (последние 5 промтов)
5. Ошибки выполнения кода отображаются понятно
