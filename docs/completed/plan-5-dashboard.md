# План реализации Этапа 5: Презентационный режим (Dashboard)

## Цель этапа
Реализовать интерфейс финального дашборда — панели презентации результатов, где пользователи могут размещать, масштабировать и настраивать визуализации для демонстрации бизнес-аналитики.

---

## Задачи этапа

### 1. Основная структура Dashboard
- [ ] Создать компонент `DashboardPanel` — правая панель приложения
- [ ] Реализовать два состояния:
  - Сворачивание (показывается только кнопка/иконка)
  - Развернутое состояние (полноценная рабочая область)
- [ ] Добавить иконку-триггер переключения
- [ ] Сохранение состояния в localStorage

### 2. Итоговая нода (Dashboard Node)
- [ ] Создать компонент `FinalDashboardNode`:
  - Принимает данные из любых `OUT` портов рабочих нод
  - Отображается как карточка на холсте с кнопкой "Добавить в дашборд"
  - Индикация подключенных входов (IN[0], IN[1]...)
- [ ] Логика сбора данных со всех подключенных портов

### 3. Конструктор дашборда
- [ ] Создать компонент `DashboardEditor`:
  - Пустая область для размещения виджетов
  - Сетка или координатная система (px-based layout)
  - Заголовок и панель управления

### 4. Drag & Drop и Resize
- [ ] Реализовать перетаскивание виджетов:
  - Возможность перетащить график из ноды в дашборд
  - Или добавить через кнопку "Добавить" из контекстного меню
- [ ] Реализовать изменение размеров виджетов (resize):
  - Курсы изменения размеров по углам
  - Минимальные/максимальные ограничения
  - Привязка к сетке (опционально)

### 5. Компоненты виджетов
- [ ] `WidgetChart` — контейнер для графика:
  - Заголовок (название источника данных)
  - Канвас рендеринга графика
  - Кнопки управления (редактировать, удалить)
- [ ] `WidgetText` — текстовый блок:
  - Редактируемое поле с HTML/Markdown разметкой
  - Выравнивание и стилизация

### 6. Система сетки и выравнивания
- [ ] Реализовать snap-to-grid при перетаскивании (шаг: например, 10px)
- [ ] Линии-хелперы для выравнивания элементов
- [ ] Автоматическое смещение соседних виджетов при перемещении

### 7. Сохранение и загрузка дашборда
- [ ] Создать интерфейс сохранения конфигурации:
  ```typescript
  interface DashboardConfig {
    id: string;
    widgets: WidgetConfig[];
    createdAt: Date;
    updatedAt: Date;
  }

  interface WidgetConfig {
    id: string;
    type: 'chart' | 'text';
    x: number;
    y: number;
    width: number;
    height: number;
    data: any; // конфигурация контента
  }
  ```
- [ ] Экспорт в JSON файл
- [ ] Импорт из JSON файла

### 8. Публикация и экспорт результатов
- [ ] Кнопка "Представить" — режим просмотра без инструментов редактирования
- [ ] Экспорт дашборда в:
  - PDF (с помощью html2pdf или puppeteer)
  - PNG/SVG изображения отдельных виджетов

### Логика Drag & Drop (без библиотек)

```typescript
const useDragDrop = () => {
  const [draggingId, setDraggingId] = useState<string | null>(null);
  const [dragOffset, setDragOffset] = useState({ x: 0, y: 0 });

  const handleMouseDown = (e: React.MouseEvent, id: string, x: number, y: number) => {
    setDraggingId(id);
    setDragOffset({ x: e.clientX - x, y: e.clientY - y });
  };

  const handleMouseMove = (e: MouseEvent) => {
    if (!draggingId || !dragContainerRef.current) return;
    
    const rect = dragContainerRef.current.getBoundingClientRect();
    const newX = e.clientX - rect.left - dragOffset.x;
    const newY = e.clientY - rect.top - dragOffset.y;
    
    updateWidgetPosition(draggingId, {
      x: Math.round(newX / 10) * 10, // Snap to 10px grid
      y: Math.round(newY / 10) * 10,
    });
  };

  return { draggingId, handleMouseDown, handleMouseMove };
};
```

## Критерии приемки Этапа 5

1. Правая панель сворачивается/разворачивается
2. Можно перетащить график из ноды в дашборд (или через кнопку)
3. Виджеты перемещаются мышкой с snap-to-grid
4. Размеры виджетов можно изменять
5. Текстовые виджеты редактируются
6. Конфигурация сохраняется в JSON и загружается корректно
7. Режим презентации (без кнопок редактирования) работает
